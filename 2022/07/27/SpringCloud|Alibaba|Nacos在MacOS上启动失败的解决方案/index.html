SpringCloud | Alibaba | Nacos 在 MacOS 上启动失败的解决方案

本文最后更新于：5 days ago


系统信息

系统：Monterey 12.5

芯片：M1

型号：MacBook Air 2020 款



可能造成启动失败的问题 及 解决方案

问题一

没有安装 Java 「或」 环境变量中没有 JAVA_HOME 「或」 JAVA_HOME 指向路径不对

解决方案 一

配置环境变量

## 去 Orcal 官网安装 Java 
sudo vim ~/.bash_profile

## 将以下内容粘贴到文件中（自己修改下相关路径）
export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk1.8.0_311.jdk/Contents/Home
export PATH=$JAVA_HOME/bin:$PATH:.:/Users/hsk/.codeexpander/installation/node_modules/.bin:/Applications/Codeexpander.app/Contents/Resources/app/app/nodes:/Applications/Codeexpander.app/Contents/Resources/app/app/node_modules/.bin

解决方案 二 | 暴力修改

修改nacos/bin/startup.sh文件，将 JAVA_HOME 直接写在里面

如下图所示，直接在 JAVA_HOME 后面添加 Java 路径就可以

可能出现的问题二

问题

权限不足

解决方案

在启动命令前面加上sudo

## 启动 Nacos
sudo sh startup.sh -m standalone

## 关闭 Nacos
sudo ./shutdown.sh
