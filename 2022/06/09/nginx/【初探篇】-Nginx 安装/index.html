【初探篇】 安装 Nginx

本文最后更新于：18 days ago

相关配置说明

服务器版本：CentOS8 来自腾讯云的轻量级服务器，大家可以去淘宝买，48/年

Nginx 版本：nginx-1.2.1.tar.gz




安装 Nginx

  • 下载 Nginx 安装包【下载链接】

  • 上传到 CentOS 任意地址（可以放在/home/your_username/下）

  • 使用下面命令解压缩

tar zxvf nginx-*.tar.gx
  • 使用下列命令进入包内（注意，版本不同请手动替换*，也可以灵活使用 tab 键）
cd nginx-*
  • 使用下列命令查看相关依赖
./configure
  • 执行下列命令（可能是将依赖放到/usr/local/nginx文件夹下，但是这个文件夹是安装 nginx 的文件夹，请大佬们解惑）
./configure --prefix=/usr/local/nginx
  • 执行以下两个命令进行安装
make
make install
  • 注意：这边可能会遇到两个问题，如如果遇到请参考下面方法，没有遇到就跳过：

配置

基础配置说明

配置文件位置 /usr/local/nginx/conf/index.conf

# 开启进程数量
worker_processes  1;

# 默认可以有多少个连接
events {
    worker_connections  1024;
}

# 
http {

		# 引入其他依赖文件 
		# mime.types 是请求头
    include       mime.types;
    
    default_type  application/octet-stream;


		# 零拷贝
    sendfile        on;
		# 保持连接，超时时间
    keepalive_timeout  65;

		# 一个主机
    server {
    		# 监听端口号
        listen       80;
        # 主机名称
        server_name  localhost;


				# url 一个主机下面可以有好几个 location 默认页
        location / {
            root   html;
            index  index.html index.htm;
        }


        #
        error_page   500 502 503 504  /50x.html;
        location = /50x.html {
            root   html;
        }


    }


  
}

Nginx 创建脚本文件

nginx.service

  • 新建一个脚本文件：/usr/lib/systemd/system/nginx.service（允许使用systemctl管理 nginx）
[Unit]
Description=nginx   -   web server
After=network.target remote-fs.target nss-lookup.target

[Service]
Type=forking
PIDFile=/usr/local/nginx/logs/nginx.pid
ExecStartPre=/usr/local/nginx/sbin/nginx -t -c /usr/local/nginx/conf/nginx.conf
ExecStart=/usr/local/nginx/sbin/nginx -c /usr/local/nginx/conf/nginx.conf
ExecReload=/usr/local/nginx/sbin/nginx -s reload
ExecStop=/url/local/nginx/sbin/nginx -s stop
ExecQuit=/usr/local/nginx/sbin/nginx -s quit
PrivateTmp=true

[Install]
WantedBy=multi-user.target

Nginx 负载均衡机制

配置文件位置/usr/local/nginx/conf/nginx.conf

方式一：轮询、权重、备用机器、下线

# 配置多个代理，名字叫 abcdefg，下方可用，可以轮询（并且配置权重）
upstream abcdefg{
	# 权重为 8 的及其
	server 110.40.191.50:80 weight=8;
	#	备用机器
  server 110.40.191.51:80 backup;
  # down 下线，不参与轮询
  server 110.40.191.52:80 down;
}


server {
        listen       80;
        server_name  localhost;


        location / {
        	# 配置代理
					proxy_pass http://abcdefg;
        }

        error_page   500 502 503 504  /50x.html;
        location = /50x.html {
            root   html;
        }
}

缺点：无法保持会话。

方式二：ip_hash

不太用（不适合当下大环境，更换 ip 之后重新 hash，那么我在旅行中的火车上一直更换移动基站，就一直 hash）

方式三 least_conn

最少连接数访问（不太常用、不太合理）

有可能因为我配置的权重较低，导致其连接数较少。

方式四：fair

根据后端服务器响应时间来转发请求（需要安装插件）

方式五： url_hash

（定向流量转发）根据 url 取 hash 值，相同的 hash 值转发到同一个服务器上

动静分离

两台服务器，其中B负责跑项目，A机器负责代理到 B 机器(B机器中不包含 css|js

|img文件)

A 机器：

方式一:

直接将需要的静态文件全部写全

/usr/local/nginx/conf/nginx.conf

location / {
	# 代理转发到 B，其中ipAddress与port分别为B中的ip以及端口
	proxy_pass ipAddress:port;
}

# A机器中的 css 文件的位置，存储在/nginx/html下（可以自己更换位置）
location /css{
  root html;
  index index.html index.htm;
}

# A机器中的 js 文件的位置，存储在/nginx/html下（可以自己更换位置）
location /js{
  root html;
  index index.html index.htm;
}

# A机器中的 img 文件的位置，存储在/nginx/html下（可以自己更换位置）
location /img{
  root html;
  index index.html index.htm;
}

方式二：正则匹配

location / {
	# 代理转发到 B，其中ipAddress与port分别为B中的ip以及端口
	proxy_pass ipAddress:port;
}

# 
location ~*(css|js|img){
  root html;
  index index.html index.htm;
}
