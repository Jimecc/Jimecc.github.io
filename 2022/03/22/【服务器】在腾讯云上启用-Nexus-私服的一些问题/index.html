解决方案

对于这个问题网上给出了很多的解决办法，但是既然给出的问题是“不能使用 root用户”，那么我的解决方案就是使用exit命令退出 root 模式后重新启动。

问题

由于 Nexus 的默认端口 8081 与开发中的 8080 比较接近，一般开发中我会将 8081 作为第二项目的启动接口（比如 Vue 等），因此我将 Nexus 的默认端口修改为了 3939。

修改方法如下：

# 执行以下命令 | 打开 nexus 的默认配置文件
vim ./etc/nexus-default.properties

# 添加以下代码后重启 Nexus
application-port=3939

问题

明明安装了 jdk1.8，配置好了环境变量，但是运行还是出现以下错误

No suitable Java Virtual Machine could be found on your system.
The version of the JVM must be at least 1.8 and at most 1.8.
Please define INSTALL4J_JAVA_HOME to point to a suitable JVM.

解决方案

在使用 vim 打开 /usr/local/nexus/nexus-3.38.0-01/bin/nexu

# 使用 vim 打开 bin 文件夹下的 nexus 文件
vim /usr/local/nexus/nexus-3.38.0-01/bin/nexus

# 添加如下代码
INSTALL4J_JAVA_HOME_OVERRIDE=/usr/local/jdk/jdk8
