泛型

泛型类

为类设置一个泛型，然后可以在类中使用它

public class Person<T>{
  private T name;
  private T address;
  
  public void setName(T name){
    this.name = name;
  }
  
  public void setAddress(T address){
    this.address = address;
  }
  public T getName(){
    return name;
  }
  public T getAddress(){
    return address;
  }
}
public class Test{
  public static void main(String[] args){
    Person<String> p = new Person<String>();
    p.setName("Test");
  }
}

泛型方法 | 普通泛型方法 | 静态方法

泛型方法在定义泛型的类型的时候可以脱离类

比如类是<T>而类可以写成<M>，然后传什么参数都可以

// 普通方法
public class Person3{
  public void show(String name){
    System.out.println(name+"正在睡觉");
  }
}

// 泛型方法
public class Person3<T>{

  // 普通泛型方法
  public void show(T name){
    System.out.println(name+"正在睡觉");
  }
  
  // 静态泛型方法
  // 静态泛型方法中的类型占位符与类中的泛型占位符没有关系
  public static <W> void show1(W name){
    System.out.println("静态方法"+name+"正在睡觉");
  }
  public static <E> E show2(E name){
  	return name;
  }
  public void show3(A name){
  	System.out.println("普通泛型方法"+name+"未指定类型");
  }
}

public class Test{
  public static void main(String[] args){
    
    // 调用普通泛型方法
    Person3<String> p = new Person3<String>();
    p.show("Test");
    
    // 调用静态泛型方法
    Person3.show1("你好世界");
    System.out.println(Person3.show2("你好世界"));
    p.show3(111);
    p.show3("Hello world");
  }
}

泛型接口

  1. 实现接口的时候可以指定具体的泛型类型
  2. 如果实现的时候没有指定，那么必须要在这个实现类中声明一个泛型给接口用
// 接口
public interface PerInt<T>{
  public void show(T name);
}

实现接口的时候直接指定泛型

// 实现类 在实现的时候定义泛型的类型
public class PerIntImpl implements PerInt<String>{
  
  @Override
  public void show(String name){
    System.out.println("Hello"+name);
  }
}

不实现相关方法

// 实现类 不指定方法
public class PerIntImpl2<T> implements PerInt<T>{
  
  @Override
  public void show(T name){
    System.out.println("Hello"+name);
  }
}

调用

public class Test{
  public static void main(String[] args){
    
    // 调用
		PerIntImpl p = new PerIntImpl();
    p.show();
    
    PerIntImpl1<String> p1 = new PerIntImpl1<String>();
  }
}

泛型擦除模式

  1. 在编码时候写的泛型类型，在反射阶段（代码编译阶段）会将泛型类型给擦除掉，所以对比以下两个泛型才会是一致的。
  2. 擦除模式就是在编码运行期间将反省全部去掉
  3. 为什么要使用擦除模式？
    1. 为了兼容 JDK 老版本的编码
public class Person4<T>(){
  private T name;
  public void setName(T name){
    this.name = name;
  }
  public T getName(){
    return name;
  }
  public void show(){
    System.out.println("方法被调用");
  }
}
public class Test{
  public static void main(String[] args){
    
		Person4<String> p  = new Person<String>();
		Person4<String> p1 = new Person<String>();    
    System.out.println(p.getClass() == p1.getClass()); // true
    
    Person4<String>  p2 = new Person<String>();
		Person4<Integer> p3 = new Person<String>();  
    System.out.println(p2.getClass() == p3.getClass()); // true
    
    // Person4 p2 = new Person();
		// Person4 p3 = new Person();  
   
  }
}

泛型通配符

泛型通配符

通配符：由于java 中的继承关系，在泛型中不做任何声明修饰的情况下是不被认可的，所以要使用通配符进行处理

通配符 一般使用 ? 来表示，可以理解为 ? 是泛型中所有类的父类

接下来会使用通配符在反省中将 java 中的继承关系重新绑定

通配符的上边界与下边界(excends 与 super )

java 中的继承并不是泛型中的继承

java 中的父子类关系在泛型中并不是父子类关系

public class Person4<T>(){
  private T name;
  
  public void setName(T name){
    this.name = name;
  }
  public T getName(){
    return name;
  }
  public void show(Person<?> p){
    this.setName((T)p.getName());
  }
}
public class Test{
  public static void main(String[] args){
    
		Person4<Number> p  = new Person4<Number>();
    Person<Integer> p1 = new Person4<Integer>();
    p1.setName(111);
    p.show(p1); // 使用 ? 之后正常，不使用 ? 会报错 
    p.show(p);  // 正常
    
    // 只有 java 中的父子关系泛型是不认的
   
  }
}

extends 与 super

? excends T 代表可以传进来 T本身 或者 T的子类（ 比如 Integer 与 Number ） 上边界
? super T 代表可以传进来 T本身 或者 T的父类 下边界

// ? excends T 代表可以传进来 T本身 或者 T的子类（ 比如 Integer 与 Number ） 上边界
// ? super T 代表可以传进来 T本身 或者 T的父类 	下边界
public class Person4<T>(){
  private T name;
  
  public void setName(T name){
    this.name = name;
  }
  public T getName(){
    return name;
  }
  
  
  // 如果是 super 的话则应该是可以传入 T 和 T 的父类
  // 表示可以传入 T 和 T 的子类
  public void show(Person<? excends T> p){
    this.setName(p.getName());
  }
}
public class Test{
  public static void main(String[] args){
    
		Person4<Number> p  = new Person4<Number>();
    Person<Integer> p1 = new Person4<Integer>();
    p1.setName(111);
    p.show(p1); // 使用 ? 之后正常，不使用 ? 会报错 
    p.show(p);  // 正常
    
    // 只有 java 中的父子关系泛型是不认的
   
  }
}

什么时候用上边界？

在读取 T 这个类型数据的时候，但不写入数据的时候，使用上边界

什么时候使用下边界？

需要写入数据但是不需要读取的时候，使用下边界
